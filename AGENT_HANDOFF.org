* Current status (2026-05-30) — read this first

The init-centering ablation drove most of the early machinery, but
the project has since absorbed several library-level refactors. Current
ground truth:

- *Tests run with* =uv run python -m pytest= (NOT =.venv/bin/python=,
  which lacks the editable install). Full suite is ~472 tests; fast suite
  (=-m "not slow"=) is ~447. See [[reference-test-invocation]] for the env
  zlib gotcha.
- *The ablation is a first-class library* =Study= (ADR-0007). It lives at
  =experiments/init_centering/study.py:INIT_CENTERING_STUDY=. The grid is
  *12 cells* (auto-degenerate clamp on parameter-free forms; not 18) ×
  *2 datasets* = *24 presets* named =init_<cell>__<dataset>= (e.g.
  =init_mlp_pinned_per_t__1d=). The presets bake the real lift datasets —
  there is no =Harmonic= placeholder and no launch-time
  =experiment.data.*= override anymore.
- *Tracking modes are now* =fixed= and =per_t=. =global_ema= was dropped
  from the ablation (the underlying =SigmaDataBuffer= still supports it as
  a valid mode; it just isn't a cell).
- *Study is library code* (ADR-0007 + 0008): =src/ddssm/study.py=
  (Study / Axis / StudyPoint) + =src/ddssm/launch.py= (PointLaunch /
  LaunchStrategy / StudyOrchestrator) + =src/ddssm/sbatch.py=.
- *One generic launcher*:
  =python -m ddssm.launch <study> [--select k=v] [--size tiny|paper|smoke]
  [--seeds ...] [--dry-run|--write-dir|--local|--submit]=. The per-family
  launchers (=launch_phase_d/ablation_tiny/paper_headline=), =launch_study=,
  and =smoke_phase_d= were *deleted*; =--local= replaces the smoke runner.
- *Grid enumeration* is =INIT_CENTERING_STUDY.names()= / =.select(...)= /
  =.points= (the old =all_phase_d_cells()= helper is gone).
- *Hparams* (=DDSSMHyperParamsConf=) hold only optimisation knobs;
  =anchor_lambda= / =lambda_sigma_p= live on the per-stage =FullELBO=
  loss objects (ADR-0004). The sweep targets
  =experiment.training.stages.{anchor_lambda,lambda_sigma_p}= — the old
  =experiment.hparams.lambda_sigma_p= axis was a silent no-op.
- *Transitions:* the legacy =GaussianInitPrior= / =ZInit= /
  =DiffusionV2Transition= paths were removed (ADR-0006). The polymorphic
  transition interface keeps =BaselineGaussianTransition= (stage 1) +
  =DiffusionV3Transition= (stage 2); =aux_posterior= is a mandatory
  constructor arg.
- *Removed/renamed files referenced in older docs no longer exist*:

* User profile + working style

- Researcher (PhD-style); comfortable reading source.
- Prefers planning *before* implementation — uses plan mode, then
  =ExitPlanMode=, then implement. Treat plan-mode review as a real
  approval gate, not a formality.
- Iterates phase-by-phase. Each phase is typically one commit. Don't
  open new PRs / push without asking.
- Asks targeted clarifying questions about design tradeoffs. Use
  =AskUserQuestion= when you'd otherwise have to guess; the user prefers
  to be consulted over Claude making executive decisions on naming, test
  depth, etc.
- Values *first-class abstractions*: when you see local dedup or repeated
  shape, prefer promoting to a library object (e.g. the =Study= refactor)
  over inlining a helper. Reuse project vocabulary from =.claude/skills/=.
- Values numerical correctness anchors. When you add abstraction, add a
  test that reduces to a known baseline (the original
  =DiffusionV2= reduction is gone, but the spirit applies — every cell
  factory should be self-consistent and verifiable).
- Values *serialize-then-plot* hygiene: any reporting code writes metrics
  to disk first so plot iteration is decoupled from computation. See
  =experiments/init_centering/report.py= for the canonical pattern.

* Environment quirks

** Test invocation

Always =uv run python -m pytest …= — bare =pytest= or
=.venv/bin/python -m pytest= will fail. =uv run= rebuilds + reinstalls
the editable =ddssm= package (src-layout + hatchling) before running.

#+begin_src bash
uv run python -m pytest tests/ -m "not slow"        # fast suite (~447)
uv run python -m pytest tests/                       # full suite (~472)
#+end_src

If =uv run python …= fails with
=ImportError: libz.so.1: cannot open shared object file=, the shell lost
the =nix develop= env (each Bash command is a fresh shell; zlib drops
off =LD_LIBRARY_PATH=). Prefix:

#+begin_src bash
export LD_LIBRARY_PATH="$(echo /nix/store/*-zlib-*/lib | tr ' ' ':'):$LD_LIBRARY_PATH"
#+end_src

or re-enter =nix develop=.

** Drive grid iteration from the Study

Every consumer that iterates the grid (the generic launcher, =report.py=,
ad-hoc analysis) goes through =INIT_CENTERING_STUDY= (=.names()= /
=.select(...)= / =.points=). To change the grid, edit the cell enumerator
(=experiments/init_centering/cells.py:iter_cells=) or the dataset axis
(=experiments/init_centering/datasets.py:ABLATION_DATASETS=) — the Study,
registration, launcher, and report all pick it up.

** Extend, don't duplicate

When you need a new launch shape (multi-node sweep, SLURM array, etc.),
add a =LaunchStrategy= in =src/ddssm/launch.py= rather than writing a
per-family launcher. The existing strategies (=single_job= /
=optuna_single_node=) + stubs (=optuna_multi_node= / =slurm_array=) show
the shape. Cross-point scheduling (cost / deadline / pool) stays in the
=plan-campaign= skill.

* Glossary

- *Cell* — one point of the 12-cell ablation grid, identified by the
  triple =(baseline_form, baseline_mode, tracking_mode)=. Named e.g.
  =init_mlp_pinned_per_t=. Now a =NamedTuple= value
  (=experiments/init_centering/cells.py:Cell=).
- *Study point* — one =(cell, dataset)= combination, registered as the
  preset =init_<cell>__<dataset>=. =INIT_CENTERING_STUDY.points= enumerates
  all 24.
- *Canonical cell* — =(mlp, pinned, per_t)=, the reference baseline.
- *Simple-smoke cell* — =(zero, pinned, fixed)=. Minimum surface +
  pipeline check. Wired as =init_smoke_simple=.
- *High-surface-smoke cell* — =(mlp, learnable, per_t)=. Exercises every
  cell-machinery code path. Wired as =init_smoke_high_surface=.
- *Handoff* — Stage-1 → stage-2 transition. Pinned mode freezes μ_p at
  handoff; Learnable mode allows continued updates via the =r_mu_p=
  anchor regulariser.
- *σ_data tracking* — =SigmaDataBuffer= records the per-t target variance
  for EDM preconditioning. The ablation studies two modes: =fixed=
  (snapshot at stage-2 start) and =per_t= (independent per-timestep EMA).
  =global_ema= is supported by the buffer but isn't in the grid.
- *Sweep knobs* — sampled by Optuna per trial (log-uniform priors,
  cannot reach zero per protocol): =n_pretrain=, =sigma_pert=, plus the
  three optimiser LRs and (per ADR-0004) =anchor_lambda= /
  =lambda_sigma_p= on the per-stage =FullELBO=.
