* How to use this log

One headline per experiment, dated. Each headline holds:

1. A Python src block that constructs the experiment (or tweaks an
   existing one from ~experiments/<name>.py~). Tangle with ~C-c C-v t~
   to write a runnable script under ~runs/~ next to the results.
2. A shell src block invoking the experiment.
3. A short results table or note: what the metric was, what changed
   versus the baseline, what to try next.

The log is append-only; never edit a finished entry, add a follow-up
headline instead. Keep entries small enough to scan in a minute.

* Worked example — harmonic + diffusion, low sigma_min

** 2026-05-12 sigma_min=0.001 vs default 2e-3

Hypothesis: a smaller noise floor sharpens the score near the data
manifold and should lower CRPS on the harmonic eval.

#+BEGIN_SRC python :tangle runs/harm_diff_low_sigma/conf.py :mkdirp yes
from experiments.harmonic_diffusion import exp as base
from ddssm.builders import Schedule
import dataclasses
exp = dataclasses.replace(
    base,
    model=dataclasses.replace(
        base.model,
        transition=dataclasses.replace(
            base.model.transition,
            schedule=Schedule(sigma_min=0.001, S_k=20),
        ),
    ),
    training=dataclasses.replace(base.training, steps=500, log_every=25),
)
