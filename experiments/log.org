* How to use this log

This file holds the running record of sweeps, tweaks, and one-off
runs. The named experiments live in [[file:../experiments.org][../experiments.org]]; that's where
new presets get tangled into ~experiments/<name>.py~. This file
imports those presets and derives variants from them via
~override()~.

One headline per entry. Inside the headline:

1. A Python src block that derives + runs the variant.
2. A results table or note.

Append-only. Never rewrite a finished entry; add a follow-up.

** The patterns

| What you want                        | How                                                                 |
|--------------------------------------+---------------------------------------------------------------------|
| Run a named experiment as-is         | ~run(exp, run_dir="...")~                                           |
| Change a scalar                      | ~override(exp, "training.steps=200")~                               |
| Change several scalars               | ~override(exp, "training.steps=200", "hparams.S=4")~                |
| Loop one knob across N values        | ~for v in [...]: override(exp, f"key={v}")~                         |
| Swap a submodule (Python object)     | ~override(exp, {"model.transition.unet": MLPUnet(channels=64)})~    |
| Combine string and dict overrides    | ~override(exp, "training.steps=42", {"model...": SomeObj()})~       |
| Promote a variant to a named preset  | Add a new headline in [[file:../experiments.org][experiments.org]] and tangle it                |
