* Overview

This file is the source of truth for every DDSSM experiment. Each
named model, dataset, and experiment is defined below in a Python
src block whose ~:tangle~ argument writes a file into
~experiments/~. Tangle with ~C-c C-v t~; the resulting modules are
picked up by the Hydra CLI, by tests, and by notebook imports.

Three hydra-zen ~store()~ groups carry the named configs:

| Group        | What lives there            | CLI                       |
|--------------+-----------------------------+---------------------------|
| ~model~      | full :class:`DDSSM` configs | ~model=small_diff~        |
| ~data~       | data modules                | ~data=harmonic~           |
| ~experiment~ | bundled (model + data + …)  | ~experiment=harmonic_gauss~ |

Train / evaluate / visualise are independent stages — run any once
you have a checkpoint:

#+BEGIN_SRC sh
python -m ddssm.app      experiment=harmonic_diffusion
python -m ddssm.evaluate experiment=harmonic_diffusion \
    +checkpoint=outputs/<run_dir>/checkpoints/ckpt_latest.pth
python -m ddssm.visualize experiment=harmonic_diffusion \
    +checkpoint=outputs/<run_dir>/checkpoints/ckpt_latest.pth
#+END_SRC

W&B is opt-in (disabled by default); enable per-run with
~wandb=enabled experiment.wandb_config.project=ddssm~.

** The store idiom

Following the hydra-zen tutorial on config groups, every named
preset lives in a pre-grouped store. The three stores are defined
once and imported everywhere:

#+BEGIN_SRC python :tangle experiments/_registry.py :mkdirp yes
"""Hydra-zen config-group stores for DDSSM.

Three groups, three stores. Every named preset registers itself with
its store at import time; ``ddssm._experiment_registry`` pushes the
union into Hydra's ConfigStore at CLI startup.

* ``model_store``      — full :class:`DDSSM` compositions (shape +
                          encoder/decoder/z_init/transition baked in).
* ``data_store``       — :class:`Synthetic` / :class:`KDD` data modules.
* ``experiment_store`` — :class:`ExperimentC` instances that tie a
                          model + dataset + training/eval/viz together.

Defined in :file:`verifications.org`; tangled here.
"""

from __future__ import annotations

from hydra_zen import store

model_store = store(group="model")
data_store = store(group="data")
experiment_store = store(group="experiment")

__all__ = ["model_store", "data_store", "experiment_store"]
#+END_SRC


Every model / dataset / experiment block below ends with a visible
~store(thing, name=…)~ call so registration shows up in source, not
in an import-time walk.

** Models

A model bakes its shape (~data_dim~, ~latent_dim~, ~j~,
~emb_time_dim~, ~covariate_dim~) into every leaf
(~encoder~/~decoder~/~z_init~/~transition~) and ships with a default
~Hparams()~. Experiments override hparams as needed via
~dataclasses.replace~ at composition time.

Eight models cover four shape families × two transitions, plus two
DiffusionV2 ``probe'' variants:

| Name           | D | d | j | et | cov | Transition     |
|----------------+---+---+---+----+-----+----------------|
| ~small_gauss~  | 1 | 4 | 1 | 16 |   0 | Gaussian       |
| ~small_diff~   | 1 | 4 | 1 | 16 |   0 | Diffusion      |
| ~probe_small~  | 1 | 4 | 1 | 16 |   0 | Diffusion-V2   |
| ~robot2d_gauss~| 2 | 6 | 2 | 16 |   0 | Gaussian       |
| ~robot2d_diff~ | 2 | 6 | 2 | 16 |   0 | Diffusion      |
| ~kdd_gauss~    | 6 | 8 | 1 | 32 |   3 | Gaussian       |
| ~kdd_diff~     | 6 | 8 | 1 | 32 |   3 | Diffusion      |
| ~probe_medium~ | 4 | 8 | 1 | 16 |   0 | Diffusion-V2   |

#+BEGIN_SRC python :tangle experiments/_models.py :mkdirp yes
"""Named :class:`DDSSM` configs — one per (shape, transition) combination.

Each model bakes shape (``data_dim``, ``latent_dim``, ``j``,
``emb_time_dim``, ``covariate_dim``) into every leaf and ships with
default :class:`Hparams`. Experiments override hparams when needed via
``dataclasses.replace`` or :func:`experiments._make.override`.

Registered under ``group="model"``. Reach them via
``python -m ddssm.app model=<name>`` or
``from experiments._models import SmallGauss``.

Defined in :file:`verifications.org`; tangled here.
"""

from __future__ import annotations

from ddssm.builders import (
    DDSSM, Decoder, DiffTransition, DiffV2Transition, Encoder,
    GaussTransition, Hparams, Schedule, ScheduleV2, Unet, ZInit,
)

from experiments._registry import model_store


# ---------------------------------------------------------------------------
# 1D synthetic (D=1, d=4, j=1, emb_time_dim=16) — covers
# {lgssm, harmonic, bimodal, bimodal-noisy}.
# ---------------------------------------------------------------------------

def _ddssm_1d(transition):
    return DDSSM(
        data_dim=1, latent_dim=4, j=1, emb_time_dim=16, covariate_dim=0,
        encoder=Encoder(data_dim=1, latent_dim=4, j=1, emb_time_dim=16,
                        covariate_dim=0, use_mask=False),
        decoder=Decoder(data_dim=1, latent_dim=4, j=1, emb_time_dim=16,
                        covariate_dim=0),
        z_init=ZInit(latent_dim=4, j=1, emb_time_dim=16, covariate_dim=0),
        transition=transition,
        hyperparams=Hparams(),
        use_observation_mask=False,
    )


SmallGauss = _ddssm_1d(GaussTransition(
    latent_dim=4, j=1, emb_time_dim=16, covariate_dim=0,
))
SmallDiff = _ddssm_1d(DiffTransition(
    latent_dim=4, j=1, emb_time_dim=16, covariate_dim=0,
    unet=Unet(), schedule=Schedule(),
))
ProbeSmall = _ddssm_1d(DiffV2Transition(
    latent_dim=4, j=1, emb_time_dim=16, covariate_dim=0,
    unet=Unet(), schedule=ScheduleV2(),
))


# ---------------------------------------------------------------------------
# 2D robot navigation (D=2, d=6, j=2, emb_time_dim=16).
# ---------------------------------------------------------------------------

def _ddssm_robot2d(transition):
    return DDSSM(
        data_dim=2, latent_dim=6, j=2, emb_time_dim=16, covariate_dim=0,
        encoder=Encoder(data_dim=2, latent_dim=6, j=2, emb_time_dim=16,
                        covariate_dim=0, use_mask=False),
        decoder=Decoder(data_dim=2, latent_dim=6, j=2, emb_time_dim=16,
                        covariate_dim=0),
        z_init=ZInit(latent_dim=6, j=2, emb_time_dim=16, covariate_dim=0),
        transition=transition,
        hyperparams=Hparams(),
        use_observation_mask=False,
    )


Robot2DGauss = _ddssm_robot2d(GaussTransition(
    latent_dim=6, j=2, emb_time_dim=16, covariate_dim=0,
))
Robot2DDiff = _ddssm_robot2d(DiffTransition(
    latent_dim=6, j=2, emb_time_dim=16, covariate_dim=0,
    unet=Unet(), schedule=Schedule(),
))


# ---------------------------------------------------------------------------
# KDD Cup 2018 (D=6, d=8, j=1, emb_time_dim=32, covariate_dim=3).
# ---------------------------------------------------------------------------

def _ddssm_kdd(transition):
    return DDSSM(
        data_dim=6, latent_dim=8, j=1, emb_time_dim=32, covariate_dim=3,
        encoder=Encoder(data_dim=6, latent_dim=8, j=1, emb_time_dim=32,
                        covariate_dim=3, use_mask=False),
        decoder=Decoder(data_dim=6, latent_dim=8, j=1, emb_time_dim=32,
                        covariate_dim=3),
        z_init=ZInit(latent_dim=8, j=1, emb_time_dim=32, covariate_dim=3),

        use_observation_mask=False,

KDDGauss = _ddssm_kdd(GaussTransition(

    latent_dim=8, j=1, emb_time_dim=32, covariate_dim=3,
    unet=Unet(), schedule=Schedule(),

# ---------------------------------------------------------------------------
# Variance-probe D=4 (nonlinear-bimodal-lift only).
# ---------------------------------------------------------------------------

ProbeMedium = DDSSM(
    data_dim=4, latent_dim=8, j=1, emb_time_dim=16, covariate_dim=0,
    encoder=Encoder(data_dim=4, latent_dim=8, j=1, emb_time_dim=16,
                    covariate_dim=0, use_mask=False),
    decoder=Decoder(data_dim=4, latent_dim=8, j=1, emb_time_dim=16,
                    covariate_dim=0),

        unet=Unet(), schedule=ScheduleV2(),
    ),
    hyperparams=Hparams(),
    use_observation_mask=False,
)


model_store(SmallGauss, name="small_gauss")
model_store(SmallDiff, name="small_diff")
model_store(ProbeSmall, name="probe_small")

| ~lgssm~                    | lgssm                    | 1 |         512 |
| ~harmonic~                 | harmonic                 | 1 |        1024 |
| ~bimodal~                  | bimodal                  | 1 |        1024 |
| ~bimodal_noisy~            | bimodal-noisy            | 1 |        1024 |
| ~robot2d~                  | robot-basis-pursuit      | 2 |        1024 |
| ~nonlinear_bimodal_lift~   | nonlinear-bimodal-lift   | 4 |         256 |
| ~probe_lgssm~              | lgssm (probe budget)     | 1 |         256 |

** Experiments

Each experiment imports a model + a dataset and calls
~experiment(...)~ from ~experiments._make~ to bundle them with
training / eval / viz scalars. The result is registered to
~experiment_store~ under its preset name.

The ~experiment()~ helper keeps ~model.hyperparams~ in sync with the
experiment-level ~hparams~ field — the trainer reads one, the DDSSM
internals read the other.

*** Synthetic 1D smoke (LGSSM)

LGSSM mode trains in a few hundred steps; useful for CI.

#+BEGIN_SRC python :tangle experiments/synthetic_gauss.py :mkdirp yes
"""LGSSM smoke test + Gaussian transition (500 steps)."""

from ddssm.builders import Eval, Hparams, Plot, Training, Viz

from experiments._make import experiment, run
from experiments._models import SmallGauss
from experiments._datasets import LGSSM
from experiments._registry import experiment_store


exp = experiment(
    data=LGSSM,
    model=SmallGauss,
    hparams=Hparams(
        S=1, batch_size=32, grad_accum_steps=1,
        lambda_schedule="cosine", lambda_start=0.001, lambda_end=1.0,
        lambda_warmup_steps=200,
        enc_lr=5e-4, dec_lr=5e-4, zinit_lr=5e-4, trans_lr=5e-4,
    ),
    training=Training(steps=500, log_every=25, amp=False),
    eval=Eval(metrics=["loss_tail", "recon_mse"], split="val"),
    viz=Viz(
        plots=[Plot(name="metrics_csv", save_filename="train_loss.png",
                    kwargs={"keys": ["loss/total"], "log_y": True})],
        split="val", num_samples=10, T_split=32,
    ),
)
experiment_store(exp, name="synthetic_gauss")


if __name__ == "__main__":
    run(exp, run_dir="runs/synthetic_gauss")
#+END_SRC

*** JSD analysis (one-step distribution match)

The bimodal DGP has a closed-form one-step conditional density (50/50 mixture
of two Gaussians at ~±step_size~ around ~a·xₜ₋₁~). We compare each model's
one-step forecast distribution against this analytic truth using the
Jensen–Shannon divergence (JSD), and contrast against an LOCF baseline that
predicts ~xₜ = xₜ₋₁~ (the worst-case unimodal-collapse scenario).

This recovers the analysis previously done in the legacy ~bimodal_jsd.py~ /
~plot_bimodal_jsd.py~ scripts.

**** 1. Compute per-model JSD (writes ~bimodal_jsd.npz~ as a side-file)

The ~bimodal_jsd~ metric is registered in the eval registry; override the
existing ~bimodal_*~ eval spec at the CLI to add it. Use ~T_split=T-1~ so the
forecast horizon is strictly one step (the analytic truth is a one-step density).

#+BEGIN_SRC sh
# Gaussian
python -m ddssm.evaluate experiment=bimodal \
    +checkpoint=runs/bimodal/gauss/checkpoints/ckpt_latest.pth \
    +csv_path=runs/bimodal/gauss/metrics.csv \
    'experiment.eval.metrics=[bimodal_jsd]' \
    'experiment.eval.num_samples=1024' \
    'experiment.eval.T_split=63' \
    'experiment.eval.kwargs={bimodal_jsd: {npz_path: bimodal_jsd.npz}}' \
    hydra.run.dir=runs/bimodal/gauss

# Diffusion
python -m ddssm.evaluate experiment=bimodal \
    transition=diffusion \
    +checkpoint=runs/bimodal/diff/checkpoints/ckpt_latest.pth \
    +csv_path=runs/bimodal/diff/metrics.csv \
    'experiment.eval.metrics=[bimodal_jsd]' \
    'experiment.eval.num_samples=1024' \
    'experiment.eval.T_split=63' \
    'experiment.eval.kwargs={bimodal_jsd: {npz_path: bimodal_jsd.npz}}' \
    hydra.run.dir=runs/bimodal/diff
#+END_SRC

The ~kwargs={bimodal_jsd: {...}}~ override is per-metric — the runner forwards
that nested dict as ~**kwargs~ when it calls ~eval_bimodal_jsd~. Each run
writes its own ~bimodal_jsd.npz~ next to ~metrics.json~.

*** Diffusion V2 vs V1 — head-to-head on bimodal

#+BEGIN_QUOTE
Comparison of the original EDM-style diffusion (~transition=diffusion~, "V1")
against the VP-SDE / explicit-score-matching variant
(~transition=diffusion_v2~, "V2") on the bimodal dataset.  V2 differs from V1
in three places relevant to this comparison:

1. Schedule: variance-preserving (~beta_min, beta_max~) on ~tau in [tau_min, 1]~
   instead of EDM's ~rho~-power ~sigma~-grid.
2. Loss: closed-form *explicit* score matching using the encoder posterior
   ~(mu_t, sigma_t**2)~ at the target step (V1 uses standard DSM around the
   sampled ~z_t~).
3. Importance sampling: ~k_sampling_mode=importance~ is currently disabled
   (raises ~NotImplementedError~) — the previous LSGM-style derivation
   ~p(tau) ∝ d(sigma_tilde**2)/dtau~ was incorrect and is being re-derived.
   Use ~k_sampling_mode=uniform~ for V2 runs in the meantime.
#+END_QUOTE

Both runs share the same encoder/decoder/zinit, the same data module, and the
same training-step budget; only the transition (and its sampling/loss) changes.
Use the *same* ~lambda_warmup_steps~ as the V1 baseline so the rate term
ramps in identically.

All training commands in this section pass ~wandb=enabled
experiment.wandb_config.group=bimodal_v1_v2~ so the per-step losses (~loss/total~, ~loss/recon~,
~loss/trans~, ~loss/zinit~, ~loss/L_p~, ~loss/L_q~, etc.) for V1 and V2 all
show up in the same W&B group and can be overlaid.  Eval and
visualize stages do not need W&B; the training metrics live in the W&B run for
each ~hydra.run.dir~.  If you want to retrain the V1 / Gaussian baselines under
the same group, prepend ~wandb=enabled experiment.wandb_config.group=bimodal_v1_v2~ to those
commands too (re-run the bimodal section above with that override).

* Appendix

** Adding a new experiment

The configuration package lives at ~src/ddssm/conf/~ and mirrors the
section structure of this document:

| ~experiments/component.py~ | Component Tests (synthetic\_gauss, synthetic\_diffusion)                        |
| ~experiments/synthetic.py~ | Synthetic Experiments (harmonic, bimodal, robot\_2d base presets) |
| ~experiments/kdd.py~       | Real-Data Experiment: KDD Cup 2018                                              |

To add a new experiment:

1. **For a new synthetic family**: add the data-generating process to
   ~src/ddssm/data/synthetic.py~, define eval/viz defaults in
   ~src/ddssm/conf/_eval_viz.py~, and register a single base preset in
   ~experiments/synthetic.py~ with ~_experiment_conf(...)~ (no
   ~transition_conf~).  All transition and training-budget variants are
   then expressed as CLI overrides in this file.
2. **For a new real-data dataset**: follow the pattern in
   ~experiments/kdd.py~.  Since dataset-specific hyper-parameters
   (batch size, steps) differ between Gaussian and Diffusion variants,
   registering one preset per transition is reasonable.
3. Register with ~store(..., group="experiment", name="my_preset")~ and
   add any new eval/viz families to ~src/ddssm/conf/_eval_viz.py~.
4. Add coverage to ~tests/test_experiment_configs.py~ and an entry in
   the Results table above.

** Adding a new metric

1. Write a function with signature ~(ctx: EvalContext) -> dict[str, Any]~.
2. Decorate with ~@register_metric("my_metric")~ in
   ~src/ddssm/eval/metrics.py~.
3. Reference it in an ~EvalSpec(metrics=["my_metric", ...])~.

** Adding a new plot

1. Write a function with signature ~(ctx: PlotContext, save_path: str, **kwargs) -> None~.
2. Decorate with ~@register_plot("my_plot")~ in ~src/ddssm/viz/plots.py~.
3. Reference it in a ~VizSpec~ ~PlotSpec(name="my_plot", ...)~.
