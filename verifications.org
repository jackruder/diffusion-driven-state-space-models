* Overview

This notebook documents component tests and synthetic-data comparisons for
DDSSM using the Hydra-based three-stage CLI. Each experiment is a named preset
in ~ddssm.conf~; train, evaluate, and visualize are independent stages that can
be run in any order provided a checkpoint exists.

** Three-stage pattern

#+BEGIN_SRC sh
# 1. Train
python -m ddssm.app experiment=PRESET

# 2. Evaluate (requires a checkpoint from stage 1)
python -m ddssm.evaluate experiment=PRESET \
    +checkpoint=outputs/<run_dir>/checkpoints/ckpt_latest.pth \
    +csv_path=outputs/<run_dir>/metrics.csv

# 3. Visualize (produces PNGs under the same run_dir)
python -m ddssm.visualize experiment=PRESET \
    +checkpoint=outputs/<run_dir>/checkpoints/ckpt_latest.pth \
    +csv_path=outputs/<run_dir>/metrics.csv
#+END_SRC

Hydra writes all outputs (CSV log, checkpoints, PNGs, ~metrics.json~) under
~outputs/<date>/<time>/~ by default.  Pass ~hydra.run.dir=runs/my_run~ to
override.

** W&B logging

#+BEGIN_SRC sh
# Enable W&B (opt-in)
python -m ddssm.app experiment=PRESET wandb=enabled wandb.project=ddssm
#+END_SRC

W&B is disabled by default (~conf/wandb/disabled.yaml~). Use ~wandb=enabled~ to
activate; group / run names are auto-derived from Hydra so multirun trials
cluster correctly in the W&B UI.

* Component Tests

These smoke tests verify that individual model components work in isolation
before running full comparison experiments.

** Encoder / Decoder — Reconstruction Only

Train only encoder, decoder, and z-initializer; transition is frozen and its
loss term is disabled. This checks whether the posterior and decoder form a
working VAE independently of the learned dynamics.

#+BEGIN_SRC sh
python -m ddssm.app experiment=synthetic_gauss \
    'experiment.training.trainable.transition=false' \
    'experiment.training.compute_trans=false' \
    'experiment.training.steps=500' \
    hydra.run.dir=runs/component/recon_only
#+END_SRC

#+RESULTS:
:

*** Expected behavior
- ~loss/recon~ decreases steadily; ~loss/trans~ stays near 0.
- Reconstruction plot shows tight overlap between observed and decoded posterior.

#+BEGIN_SRC sh
python -m ddssm.visualize experiment=synthetic_gauss \
    +checkpoint=runs/component/recon_only/checkpoints/ckpt_latest.pth \
    +csv_path=runs/component/recon_only/metrics.csv \
    hydra.run.dir=runs/component/recon_only
#+END_SRC

** Bimodal — Multimodality Comparison

#+BEGIN_QUOTE
Key experiment.  The bimodal data-generating process performs a random-walk
with sign-alternating jumps (~z_t = 0.9 z_{t-1} + 4 sₜ~, ~sₜ ∈ {-1, +1}~).
The forecast distribution has two well-separated modes at each horizon; a
unimodal Gaussian transition cannot capture this.  The diffusion transition
should produce measurably lower energy score.
#+END_QUOTE

unimodal predictive distributions regardless of their location.  CRPS is

* Appendix

** Adding a new synthetic mode

1. Implement the data-generating process in ~src/ddssm/data/synthetic.py~
   (add a branch to the ~mode~ dispatch in ~SyntheticDataset.__init__~).
2. Add a ~SyntheticDataModuleConf(mode="my_mode", ...)~ and a corresponding
   experiment conf in ~src/ddssm/conf.py~, wiring eval/viz confs as shown
   for the harmonic/bimodal presets.
3. Register with ~store(..., group="experiment", name="my_mode_gauss")~.

** Adding a new metric

1. Write a function with signature ~(ctx: EvalContext) -> dict[str, Any]~.
2. Decorate with ~@register_metric("my_metric")~ in
   ~src/ddssm/eval/metrics.py~.
3. Reference it in an ~EvalSpec(metrics=["my_metric", ...])~.

** Adding a new plot

1. Write a function with signature ~(ctx: PlotContext, save_path: str, **kwargs) -> None~.
2. Decorate with ~@register_plot("my_plot")~ in ~src/ddssm/viz/plots.py~.
3. Reference it in a ~VizSpec~ ~PlotSpec(name="my_plot", ...)~.
