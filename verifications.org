* Overview

This notebook documents component tests and synthetic-data comparisons
for DDSSM. Each experiment is a plain Python file under
~experiments/~ in the repo root that calls ~make_experiment(...)~
from ~experiments/_make.py~ to compose an ~Experiment~. The Hydra CLI
auto-discovers every ~experiments/<name>.py~ at startup, so
~experiment=NAME~ Just Works.

Train, evaluate, and visualize are independent stages — run any of
them once you have a checkpoint.

** Where experiments live

#+BEGIN_SRC text
experiments/
    _make.py                       # make_experiment, run, to_yaml, tweak
    _bases.py                      # one base() function per dataset family
    harmonic_gauss.py              # one variant per file (3-5 lines each)
    harmonic_diffusion.py
    bimodal_gauss.py
    bimodal_diffusion.py
    robot_2d_gauss.py
    robot_2d_diffusion.py
    synthetic_gauss.py
    synthetic_diffusion.py
    kdd_gauss.py
    kdd_diffusion.py
    variance_probe_*.py
    log.org                        # the experiment log
#+END_SRC

Each *base* in ~_bases.py~ (~harmonic_base~, ~bimodal_base~, …) is a
function that takes the variable slot (~transition~) plus a few
training/hparams knobs that diverge between Gaussian and diffusion
variants. Each *variant file* is a 3-5 line diff:

#+BEGIN_SRC python
# experiments/harmonic_gauss.py
from ddssm.builders import GaussTransition
from experiments._bases import harmonic_base
exp = harmonic_base(GaussTransition())

# experiments/harmonic_diffusion.py
from ddssm.builders import DiffTransition, Schedule, Unet
from experiments._bases import harmonic_base
exp = harmonic_base(
    DiffTransition(unet=Unet(), schedule=Schedule()),
    steps=2000, checkpoint_every=500, lambda_warmup_steps=400,
)
#+END_SRC

** Tweaking a single parameter

Use ~tweak()~ for nested overrides via ~__~-separated paths. The
original ~exp~ is not mutated; ~tweak~ returns a fresh dataclass.

#+BEGIN_SRC python
from experiments.harmonic_diffusion import exp as base
from experiments._make import tweak, run

# One knob deep in the schedule:
exp = tweak(base, model__transition__schedule__sigma_min=0.001)

# Several at once:
exp = tweak(base,
    training__steps=500,
    hparams__lambda_warmup_steps=100,
    model__transition__schedule__sigma_min=0.001,
)
run(exp, run_dir="runs/harm_diff_sigma_low")
#+END_SRC

** Swapping CSDI for MLP variants

Each ablation network ships as a separate builder; pass it in
directly. No store(), no group override:

#+BEGIN_SRC python
from ddssm.builders import DiffTransition, MLPUnet, MLPContext, Unet
from experiments._bases import harmonic_base

# Replace the diffusion U-Net with the MLP ablation:
exp = harmonic_base(
    DiffTransition(unet=MLPUnet(channels=64, n_layers=2)),
)

# Or swap the encoder/decoder/z_init context producer for MLPs by
# passing a custom transition AND custom slots — see "Direct builds()"
# below.
#+END_SRC

Or apply to a derived experiment:

#+BEGIN_SRC python
from experiments.harmonic_diffusion import exp as base
from experiments._make import tweak
from ddssm.builders import MLPUnet
exp = tweak(base, model__transition__unet=MLPUnet(channels=32, n_layers=3))
#+END_SRC

** Direct ~builds()~ — no ~make_experiment~ at all

~make_experiment~ is convenience; you can also assemble an experiment
by hand. The shape kwargs just appear at every slot:

#+BEGIN_SRC python
from ddssm.builders import (
    DDSSM, Decoder, DiffTransition, Encoder, ExperimentC, Hparams,
    Schedule, Synthetic, Training, TrainerPartial, Unet, ZInit,
)

D, d, j, et = 1, 4, 1, 16  # shapes in one place

exp = ExperimentC(
    data=Synthetic(mode="harmonic", T=64),
    model=DDSSM(
        data_dim=D, latent_dim=d, j=j, emb_time_dim=et,
        encoder=Encoder(data_dim=D, latent_dim=d, j=j, emb_time_dim=et,
                        covariate_dim=0, use_mask=False),
        decoder=Decoder(data_dim=D, latent_dim=d, j=j, emb_time_dim=et,
                        covariate_dim=0),
        z_init=ZInit(latent_dim=d, j=j, emb_time_dim=et, covariate_dim=0),
        transition=DiffTransition(latent_dim=d, j=j, emb_time_dim=et,
                                  covariate_dim=0,
                                  unet=Unet(), schedule=Schedule()),
        hyperparams=Hparams(),
    ),
    build_trainer=TrainerPartial(),
    training=Training(steps=1000),
    hparams=Hparams(),
)
#+END_SRC

This is exactly what ~make_experiment~ generates, with the shape
kwargs threaded by hand. Use this when you want the full thing on
screen at once — e.g. one-off notebook experiments.

** Adding a brand-new experiment

Drop ~experiments/foo.py~ exposing ~exp~:

#+BEGIN_SRC python
from ddssm.builders import DiffTransition, Schedule, Unet
from experiments._bases import harmonic_base
from experiments._make import run, tweak

exp = harmonic_base(
    DiffTransition(unet=Unet(channels=128, n_layers=6),
                   schedule=Schedule(sigma_min=0.001, S_k=20)),
    steps=3000, lambda_warmup_steps=600,
)

if __name__ == "__main__":
    run(exp, run_dir="runs/foo")
#+END_SRC

No ~store()~ calls, no config groups, no presets. ~python -m
ddssm.app experiment=foo~ now works.

** Migration note (old → new experiment names)

The old single-transition presets (e.g. ~experiment=harmonic
transition=diffusion~) are now split: pick the right variant directly.

| Old CLI                                 | New CLI                          |
|-----------------------------------------+----------------------------------|
| ~experiment=harmonic~                   | ~experiment=harmonic_gauss~      |
| ~experiment=harmonic transition=diff~   | ~experiment=harmonic_diffusion~  |
| ~experiment=bimodal~                    | ~experiment=bimodal_gauss~       |
| ~experiment=bimodal transition=diff~    | ~experiment=bimodal_diffusion~   |
| ~experiment=robot_2d~                   | ~experiment=robot_2d_gauss~      |
| ~experiment=robot_2d transition=diff~   | ~experiment=robot_2d_diffusion~  |

Architectural ablations (~unet=mlp~, ~context=mlp~, ~time_mixer=...~,
~feature_mixer=...~) no longer come from config groups — set them on
the model parts directly inside the experiment file, or override
their nested fields on the CLI (e.g.
~experiment.model.transition.unet.channels=128~).

Below sections may still reference the old transition= flag pattern;
substitute the table above when running.

Hydra writes all outputs (CSV log, checkpoints, PNGs, ~metrics.json~) under
~outputs/<date>/<time>/~ by default.  Pass ~hydra.run.dir=runs/my_run~ to
override.

** Bimodal — Multimodality Comparison

*** JSD analysis (one-step distribution match)

The bimodal DGP has a closed-form one-step conditional density (50/50 mixture
of two Gaussians at ~±step_size~ around ~a·xₜ₋₁~). We compare each model's
one-step forecast distribution against this analytic truth using the
Jensen–Shannon divergence (JSD), and contrast against an LOCF baseline that
predicts ~xₜ = xₜ₋₁~ (the worst-case unimodal-collapse scenario).

This recovers the analysis previously done in the legacy ~bimodal_jsd.py~ /
~plot_bimodal_jsd.py~ scripts.

**** 1. Compute per-model JSD (writes ~bimodal_jsd.npz~ as a side-file)

The ~bimodal_jsd~ metric is registered in the eval registry; override the
existing ~bimodal_*~ eval spec at the CLI to add it. Use ~T_split=T-1~ so the
forecast horizon is strictly one step (the analytic truth is a one-step density).

#+BEGIN_SRC sh
# Gaussian
python -m ddssm.evaluate experiment=bimodal \
    +checkpoint=runs/bimodal/gauss/checkpoints/ckpt_latest.pth \
    +csv_path=runs/bimodal/gauss/metrics.csv \
    'experiment.eval.metrics=[bimodal_jsd]' \
    'experiment.eval.num_samples=1024' \
    'experiment.eval.T_split=63' \
    'experiment.eval.kwargs={bimodal_jsd: {npz_path: bimodal_jsd.npz}}' \
    hydra.run.dir=runs/bimodal/gauss

# Diffusion
python -m ddssm.evaluate experiment=bimodal \
    transition=diffusion \
    +checkpoint=runs/bimodal/diff/checkpoints/ckpt_latest.pth \
    +csv_path=runs/bimodal/diff/metrics.csv \
    'experiment.eval.metrics=[bimodal_jsd]' \
    'experiment.eval.num_samples=1024' \
    'experiment.eval.T_split=63' \
    'experiment.eval.kwargs={bimodal_jsd: {npz_path: bimodal_jsd.npz}}' \
    hydra.run.dir=runs/bimodal/diff
#+END_SRC

The ~kwargs={bimodal_jsd: {...}}~ override is per-metric — the runner forwards
that nested dict as ~**kwargs~ when it calls ~eval_bimodal_jsd~. Each run
writes its own ~bimodal_jsd.npz~ next to ~metrics.json~.

*** Diffusion V2 vs V1 — head-to-head on bimodal

#+BEGIN_QUOTE
Comparison of the original EDM-style diffusion (~transition=diffusion~, "V1")
against the VP-SDE / explicit-score-matching variant
(~transition=diffusion_v2~, "V2") on the bimodal dataset.  V2 differs from V1
in three places relevant to this comparison:

1. Schedule: variance-preserving (~beta_min, beta_max~) on ~tau in [tau_min, 1]~
   instead of EDM's ~rho~-power ~sigma~-grid.
2. Loss: closed-form *explicit* score matching using the encoder posterior
   ~(mu_t, sigma_t**2)~ at the target step (V1 uses standard DSM around the
   sampled ~z_t~).
3. Importance sampling: ~k_sampling_mode=importance~ is currently disabled
   (raises ~NotImplementedError~) — the previous LSGM-style derivation
   ~p(tau) ∝ d(sigma_tilde**2)/dtau~ was incorrect and is being re-derived.
   Use ~k_sampling_mode=uniform~ for V2 runs in the meantime.
#+END_QUOTE

Both runs share the same encoder/decoder/zinit, the same data module, and the
same training-step budget; only the transition (and its sampling/loss) changes.
Use the *same* ~lambda_warmup_steps~ as the V1 baseline so the rate term
ramps in identically.

All training commands in this section pass ~wandb=enabled
experiment.wandb_config.group=bimodal_v1_v2~ so the per-step losses (~loss/total~, ~loss/recon~,
~loss/trans~, ~loss/zinit~, ~loss/L_p~, ~loss/L_q~, etc.) for V1 and V2 all
show up in the same W&B group and can be overlaid.  Eval and
visualize stages do not need W&B; the training metrics live in the W&B run for
each ~hydra.run.dir~.  If you want to retrain the V1 / Gaussian baselines under
the same group, prepend ~wandb=enabled experiment.wandb_config.group=bimodal_v1_v2~ to those
commands too (re-run the bimodal section above with that override).

* Appendix

** Adding a new experiment

The configuration package lives at ~src/ddssm/conf/~ and mirrors the
section structure of this document:

| ~experiments/component.py~ | Component Tests (synthetic\_gauss, synthetic\_diffusion)                        |
| ~experiments/synthetic.py~ | Synthetic Experiments (harmonic, bimodal, robot\_2d base presets) |
| ~experiments/kdd.py~       | Real-Data Experiment: KDD Cup 2018                                              |

To add a new experiment:

1. **For a new synthetic family**: add the data-generating process to
   ~src/ddssm/data/synthetic.py~, define eval/viz defaults in
   ~src/ddssm/conf/_eval_viz.py~, and register a single base preset in
   ~experiments/synthetic.py~ with ~_experiment_conf(...)~ (no
   ~transition_conf~).  All transition and training-budget variants are
   then expressed as CLI overrides in this file.
2. **For a new real-data dataset**: follow the pattern in
   ~experiments/kdd.py~.  Since dataset-specific hyper-parameters
   (batch size, steps) differ between Gaussian and Diffusion variants,
   registering one preset per transition is reasonable.
3. Register with ~store(..., group="experiment", name="my_preset")~ and
   add any new eval/viz families to ~src/ddssm/conf/_eval_viz.py~.
4. Add coverage to ~tests/test_experiment_configs.py~ and an entry in
   the Results table above.

** Adding a new metric

1. Write a function with signature ~(ctx: EvalContext) -> dict[str, Any]~.
2. Decorate with ~@register_metric("my_metric")~ in
   ~src/ddssm/eval/metrics.py~.
3. Reference it in an ~EvalSpec(metrics=["my_metric", ...])~.

** Adding a new plot

1. Write a function with signature ~(ctx: PlotContext, save_path: str, **kwargs) -> None~.
2. Decorate with ~@register_plot("my_plot")~ in ~src/ddssm/viz/plots.py~.
3. Reference it in a ~VizSpec~ ~PlotSpec(name="my_plot", ...)~.
