* Overview

Verifications are authored as Python source blocks around Hydra-Zen config
objects from ~ddssm.conf.experiments.*~.  Train, evaluation, visualization, and
variance probing are intentionally separate source blocks: rerun only the stage
you are changing.  For example, if you tweak plots in ~experiment.viz~, rerun the
visualization block against the existing checkpoint instead of retraining.

Each stage writes under its ~run_dir~:

- ~resolved_config.yaml~: fully resolved config for reproducibility.
- ~experiment_log.json~: stage name, config identity, overrides, resolved config
  path, run directory, and key metrics.
- stage outputs such as ~metrics.csv~, checkpoints, ~metrics.json~, PNGs, or
  variance probe summaries.

The Hydra CLI remains available for sweeps and batch jobs.  These source blocks
are the canonical interactive verification workflow.

    BimodalExperimentConf,

    Robot2DExperimentConf,

    VarianceProbeBimodalNoisyExperimentConf,

    ConfigGroups,
    RunMetadata,
    evaluate_config,
    train_config,
    variance_config,
    visualize_config,
)


GAUSSIAN = ConfigGroups(transition=TransitionGaussianConf)
DIFFUSION = ConfigGroups(transition=TransitionDiffusionConf)
DIFFUSION_V2 = ConfigGroups(transition=TransitionDiffusionV2Conf)


@dataclass(frozen=True)
class _RunSpec:
    config: object
    run_dir: str
    identity: str
    groups: ConfigGroups | None = None
    updates: Mapping[str, object] | None = None
    tags: tuple[str, ...] = ()

    @property
    def metadata(self) -> RunMetadata:
        return RunMetadata(config_identity=self.identity)

    @property
    def checkpoint_path(self) -> str:
        return str(Path(self.run_dir) / "checkpoints" / "ckpt_latest.pth")

    @property
    def csv_path(self) -> str:
        return str(Path(self.run_dir) / "metrics.csv")
#+END_SRC

* Experiment presets quick reference

| Config object                                | Dataset / mode               | Default use |
|----------------------------------------------+------------------------------+-------------|
| ~SyntheticGaussExperimentConf~               | synthetic LGSSM              | component smoke test |
| ~SyntheticDiffusionExperimentConf~           | synthetic LGSSM              | component smoke test |
| ~HarmonicExperimentConf~                     | harmonic sine waves          | Gaussian or diffusion via ~ConfigGroups~ |
| ~BimodalExperimentConf~                      | bimodal random walk          | multimodality comparison |
| ~Robot2DExperimentConf~                      | robot-basis-pursuit          | 2D spatial forecast |
| ~KDDGaussExperimentConf~                     | KDD Cup 2018                 | real-data Gaussian |
| ~KDDDiffusionExperimentConf~                 | KDD Cup 2018                 | real-data diffusion |
| ~VarianceProbe*ExperimentConf~               | synthetic probe datasets     | variance diagnostics |

** Bimodal JSD analysis

The ~bimodal_jsd~ metric writes ~bimodal_jsd.npz~ in the run directory.  This is
an evaluation-only rerun against existing checkpoints.

    "experiment": {

            "num_samples": 1024,
            "T_split": 63,

        csv_path=run.csv_path,

* Adding new verification experiments

1. Decide whether the experiment needs only a new ~_RunSpec~ in this document or
   a reusable preset in ~src/ddssm/conf/experiments/*.py~.
2. For reusable presets, compose the preset with ~build_experiment_conf(...)~ and
   register it with the shared config store.
3. Add composition coverage to ~tests/test_experiment_configs.py~.
4. Keep train/evaluate/visualize/probe blocks separate so plotting and metrics
   can be iterated without rerunning training.
