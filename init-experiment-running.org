#+OPTIONS: toc:3 num:t

This is the operator's manual for the init-centering ablation grid.

*Status (2026-05-29):* the ablation is now a first-class **Study**, library
code (=src/ddssm/study.py= + =src/ddssm/launch.py=; ADR-0007/0008) — *12 cells
× 2 datasets = 24* presets named =init_<cell>__<dataset>= (e.g.
=init_mlp_pinned_per_t__1d=), each baking the real dataset. Run/launch via the
single generic launcher:

: python -m ddssm.launch init_centering [--select baseline_form=mlp dataset=mv] \
:     [--size tiny|paper|smoke] [--seeds 0 1] [--dry-run|--write-dir DIR|--local] [--submit]

The per-family launchers (=launch_phase_d/ablation_tiny/paper_headline=) and
=smoke_phase_d= were *deleted* — =--local= is the smoke runner; =--size paper=
is the paper-headline (2× latent) variant. Grid iteration is
=INIT_CENTERING_STUDY.names()=/=.select(...)=. The bare =experiment=init_<cell>=
commands below now need a =__<dataset>= suffix; some examples below predate the
Study refactor.

It covers:

- the project lineage A → E that landed the grid and its reporter,
- every model knob exposed at the CLI (the 18-cell axes, sweep
  knobs, NN shape, schedule, training scalars),
- four ways to run the experiments (single local, local sweep, SLURM
  sweep, report-only), and
- a one-command =smoke_phase_d= recipe that runs every cell on tiny
  data with ~5 training steps each, all the way through to a plot.

If you just want to verify the pipeline end-to-end, jump straight to
[[#sec-smoke][§5 The end-to-end smoke pass]].

* Model settings — what each knob does
:PROPERTIES:
:CUSTOM_ID: sec-model-settings
:END:

The ablation grid lives in =experiments/init_centering/=.  The shape
of a single run is determined by three nested config objects.

** The 18-cell grid (3 axes)

Source of truth: =experiments/init_centering/cells.py:iter_cells=.

| Axis            | Values                                | Default ("canonical") |
|-----------------+---------------------------------------+-----------------------|
| =baseline_form= | =zero=, =identity=, =linear=, =mlp=   | =mlp=                 |
| =baseline_mode= | =pinned=, =learnable=                 | =pinned=              |
| =tracking_mode= | =fixed=, =global_ema=, =per_t=        | =per_t=               |

Total = 4 × 2 × 3 = 24 raw triples, but =zero= and =identity= are
parameter-free so they auto-clamp =learnable → pinned= (see
=experiments/init_centering/model.py:_PARAM_FREE_FORMS=).  Net = *18
distinct cells*.

Each cell has its own named experiment preset.  Naming:
=init_<form>_<mode>_<tracking>=, e.g. =init_zero_pinned_fixed=,
=init_mlp_learnable_per_t=.

The "simplest cell" =init_zero_pinned_fixed= is mathematically
equivalent to plain =DiffusionV2= (no centering, no σ_data
tracking).  This is locked down in
=tests/test_init_centering_v2_reduction.py= via a numerical
=_vp_precondition= match to ~1e-4.

** The 2 control cells

Both pin the *canonical* cell and zero one sweep knob — values
Optuna's log-uniform cannot sample.

| Preset name                       | What it pins                                     |
|-----------------------------------+--------------------------------------------------|
| =init_canonical_ctrl_sigma0=      | =sigma_pert=0= on the canonical cell             |
| =init_canonical_ctrl_npretrain0=  | =n_pretrain=0= (no stage-1 pretrain)             |

These are run as single jobs, not Optuna sweeps.

** Per-cell model knobs

Set by =_build_init_centering_model= (=experiments/init_centering/model.py=).
All are overridable at the CLI via =experiment.model.<name>=.

| Knob                    | Default | Meaning                                                    |
|-------------------------+---------+------------------------------------------------------------|
| =j=                     | 1       | Latent history window (transition input is =z_{t-j:t-1}=)  |
| =T_max=                 | 32      | Max latent timestep (sigma_data buffer length)             |
| =data_dim=              | 1       | Observation dim                                            |
| =latent_dim=            | 4       | Latent state dim =d=                                       |
| =emb_time_dim=          | 16      | Time-embedding feature dim                                 |
| =channels=              | 32      | CSDIUnet channel width                                     |
| =nheads=                | 4       | Attention heads (must divide =channels=)                   |
| =baseline_hidden_dim=   | 32      | μ_p MLP hidden width                                       |
| =baseline_n_layers=     | 2       | μ_p MLP depth                                              |
| =anchor_lambda=         | =None=  | R_μp strength.  =None= → 0 (pinned) / 1e-2 (learnable)     |
| =diffusion_S_k=         | 1       | MC samples for ESM target                                  |
| =diffusion_k_chunk=     | 1       | k-grouping for the EDM loss                                |
| =diffusion_num_steps=   | 50      | Discrete VP-SDE timesteps                                  |
| =diffusion_layers=      | 2       | CSDIUnet residual layers                                   |

** Per-cell stage knobs (the Optuna sweep targets)

Set by =_build_init_centering_stages= (=experiments/init_centering/hparams.py=).
Override path is =experiment.model.stages.<name>=.

| Knob                  | Default | Sweep range (Phase C/D)                |
|-----------------------+---------+----------------------------------------|
| =n_pretrain=          | 200     | =tag(log, int(interval(50, 2000)))=    |
| =n_stage2=            | 600     | not swept                              |
| =sigma_pert=          | 1e-2    | =tag(log, interval(1e-4, 1e-1))=       |
| =enc_lr=              | 5e-4    | =tag(log, interval(1e-5, 1e-3))=       |
| =dec_lr=              | 5e-4    | =tag(log, interval(1e-5, 1e-3))=       |
| =trans_lr=            | 5e-4    | =tag(log, interval(1e-5, 1e-3))=       |
| =log_every=           | 25      | not swept                              |
| =checkpoint_every=    | 200     | not swept                              |
| =early_stop_enabled=  | False   | not swept                              |

The sweep search space lives at
=experiments/init_centering/sweeps.py:InitPilot= (registered as
=+sweep=init_pilot=).

** Hparams (lr / batch / λ schedule)

=experiments/init_centering/hparams.py:SmokeHparams= sets defaults
applied to *every* cell.  Most useful overrides:

| Knob                          | Default      | Notes                                           |
|-------------------------------+--------------+-------------------------------------------------|
| =hparams.S=                   | 1            | MC samples for the encoder posterior            |
| =hparams.batch_size=          | 16           |                                                 |
| =hparams.lambda_warmup_steps= | 50           | Linear-up steps for the rate-distortion λ       |
| =hparams.lambda_sigma_p=      | 1e-2         | Stage-1 log-variance anchor strength            |

** Eval + objective

| Object             | Where it lives                                                                    |
|--------------------+-----------------------------------------------------------------------------------|
| =PilotEval=        | =experiments/init_centering/evals.py= — the five Phase-A headline metrics         |
| =PilotObjective=   | =experiments/init_centering/evals.py= — reads =stage2_elbo_surrogate= from JSON   |

Eval metrics:

1. =stage2_elbo_surrogate= — the Optuna objective (minimised).
2. =sigma_data_drift= — full σ_data²(t) trajectory + 2-component
   decomposition; per-cell line in the Phase-E plot.
3. =wallclock_to_target= — first step that reached a threshold loss.
4. =crps_sum_latent= — sample-vs-GT CRPS on the latent path.
5. =gt_latent_jsd= — JSD vs GT latent kernel (LGSSM datasets only;
   Harmonic data sets =available=False=).

* Run modes
:PROPERTIES:
:CUSTOM_ID: sec-run-modes
:END:

Four ways to run.  All produce =metrics.json= + the standard Phase-A
artifacts; the reporter (=python -m experiments.init_centering.report=)
consumes them uniformly.

** Mode 1 — single local run (one cell, one trial)

#+begin_src bash
uv run python -m ddssm.app experiment=init_mlp_pinned_per_t
#+end_src

Defaults: 200 stage-1 steps + 600 stage-2 steps + 5 eval metrics.
The Hydra run dir defaults to =outputs/<date>/<time>/=.

Override any knob at the CLI:

#+begin_src bash
uv run python -m ddssm.app experiment=init_mlp_pinned_per_t \
    experiment.model.stages.n_pretrain=50 \
    experiment.model.stages.sigma_pert=5e-3 \
    hydra.run.dir=runs/scratch/quick_run
#+end_src

** Mode 2 — local Optuna sweep on one cell (no SLURM)

#+begin_src bash
uv run python -m ddssm.app --multirun \
    experiment=init_mlp_pinned_per_t \
    +sweep=init_pilot \
    hydra.sweeper.n_trials=20 \
    hydra.sweeper.study_name=local_$(date +%s) \
    hydra.sweep.dir=runs/sweeps/local_mlp_pinned_per_t
#+end_src

Runs sequentially (=n_jobs=1=).  Each trial lands at
=runs/sweeps/local_mlp_pinned_per_t/<trial>/=.

** Mode 3 — SLURM sweep over all 18 cells + 2 controls

The Phase-D launcher generates one sbatch script per cell:

#+begin_src bash
# 1) Render scripts (does NOT submit)
uv run python -m experiments.init_centering.launch_phase_d \
    --write-dir runs/sbatch/phase_d_$(date +%Y%m%d) \
    --study-prefix phase_d_$(date +%Y%m%d) \
    --n-trials 20

# 2) Submit them
for f in runs/sbatch/phase_d_*/*.sbatch; do sbatch "$f"; done
#+end_src

Default resources (per cell): =partition=gpu=, =time=04:00:00=,
=gpus=1=, =cpus=4=, =mem=32G=.  Override at the CLI:

#+begin_src bash
uv run python -m experiments.init_centering.launch_phase_d \
    --write-dir runs/sbatch/overnight \
    --time 12:00:00 --mem 64G --n-trials 40
#+end_src

The launcher pins =hydra.sweep.dir= per cell to
=runs/sweeps/{study_prefix}_{cell}/= so the reporter can pick the
runs up unchanged.  Controls (=init_canonical_ctrl_*=) submit as
single (non-multirun) jobs.

Sanity-check before submitting:

#+begin_src bash
# Print all 20 scripts to stdout, never touches disk
uv run python -m experiments.init_centering.launch_phase_d --dry-run

# Just one cell's script
uv run python -m experiments.init_centering.launch_phase_d \
    --cell init_mlp_pinned_per_t --dry-run
#+end_src

** Mode 4 — report (no training; just aggregate + plot)

Run once the sweeps are complete:

#+begin_src bash
# Aggregate + plot in one go
uv run python -m experiments.init_centering.report all \
    --sweeps-root runs/sweeps \
    --optuna-dir runs/optuna \
    --study-prefix phase_d_$(date +%Y%m%d) \
    --out runs/report/phase_d
#+end_src

Outputs:

| Artifact                                  | Format | Contents                                              |
|-------------------------------------------+--------+-------------------------------------------------------|
| =runs/report/.../summary.csv=             | CSV    | One row / (cell, trial); 17 scalar columns           |
| =runs/report/.../records.jsonl=           | JSONL  | Full records incl. σ_data²(t) + per-t trajectories   |
| =runs/report/.../plots/sigma_data_drift.png= | PNG | One line per cell, σ_data²(t) avg across trials      |
| =runs/report/.../plots/wallclock_to_target.png= | PNG | Bar chart, mean ± std across trials                  |
| =runs/report/.../plots/headline_table.md=   | MD   | 5 metrics × N cells, best trial per cell             |

Iterate on plot code without re-aggregating:

#+begin_src bash
uv run python -m experiments.init_centering.report plot \
    --in runs/report/phase_d \
    --out runs/report/phase_d/plots_v2
#+end_src

The =plot= subcommand reads only =records.jsonl= — no model loaded,
no Optuna DB touched, no sweep dirs walked.

* The end-to-end smoke pass
:PROPERTIES:
:CUSTOM_ID: sec-smoke
:END:

Goal: in one command, prove the full pipeline (factory → training →
eval → metrics.json → aggregation → plotting) works for every cell
in the grid.

#+begin_src bash
uv run python -m experiments.init_centering.smoke_phase_d --report
#+end_src

Defaults:

- Steps per stage: =5= (stage 1 + stage 2 each).
- All 20 cells (18 grid + 2 controls), run sequentially on CPU.
- Output dir: =runs/smoke_phase_d/=.
- Hooked to the Phase-E reporter so you get the plots at the end.

Expected wall time: ~30 minutes on CPU (each cell ~90s, mostly eval).

** What it does

1. Iterates every cell of =all_phase_d_cells()= (the same list the
   sbatch launcher uses).
2. Runs each as a single =python -m ddssm.app experiment=<cell>=
   call with shrunk stage budgets:
   =experiment.model.stages.n_pretrain=5=,
   =experiment.model.stages.n_stage2=5=,
   =experiment.model.stages.log_every=1=,
   =experiment.model.stages.checkpoint_every=100=.
3. Pins =hydra.run.dir= to
   =runs/smoke_phase_d/sweeps/smoke_phase_d_{cell}/= so the reporter
   can locate the runs.
4. (=--report=) calls
   =python -m experiments.init_centering.report all= against the
   smoke outputs and writes =runs/smoke_phase_d/report/=.

** Targeted smoke

Just one cell to debug a specific regression:

#+begin_src bash
uv run python -m experiments.init_centering.smoke_phase_d \
    --cell init_zero_pinned_fixed --report
#+end_src

Continue past failures (default is fail-fast):

#+begin_src bash
uv run python -m experiments.init_centering.smoke_phase_d \
    --no-stop-on-error --report
#+end_src

** Verifying the smoke succeeded

After the script finishes, the three things to inspect:

#+begin_src bash
# 1) Did every cell produce a metrics.json?
ls runs/smoke_phase_d/sweeps/smoke_phase_d_*/metrics.json | wc -l   # expect 20

# 2) Does the headline table parse?
cat runs/smoke_phase_d/report/plots/headline_table.md

# 3) Eyeball the σ_data drift plot
xdg-open runs/smoke_phase_d/report/plots/sigma_data_drift.png
#+end_src

* Output layout cheat-sheet
:PROPERTIES:
:CUSTOM_ID: sec-outputs
:END:

Each =python -m ddssm.app= invocation produces this layout in its
=hydra.run.dir=:

#+begin_example
<run_dir>/
├── .hydra/                         # Hydra-saved config + overrides
├── resolved_config.yaml            # Fully-resolved config (incl. defaults)
├── metrics.csv                     # Per-step training metrics (loss, λ, σ_data² per-t, ...)
├── metrics.json                    # Phase-A eval results (5 headline metrics)
├── checkpoints/
│   ├── ckpt_step_<N>.pth           # Periodic (every checkpoint_every steps)
│   └── ckpt_final.pth              # Saved at end of training; what the eval loads
├── tb_logs/                        # TensorBoard
└── ddssm_app.log                   # Hydra logger output
#+end_example

For a *multirun*, the per-trial directories sit under
=hydra.sweep.dir=:

#+begin_example
runs/sweeps/phase_d_init_mlp_pinned_per_t/
├── 0/                              # trial 0 (full <run_dir> contents above)
├── 1/                              # trial 1
├── …
├── 19/                             # trial 19
└── multirun.yaml
#+end_example

The corresponding Optuna DB at
=runs/optuna/phase_d_init_mlp_pinned_per_t.db= holds trial values +
sampled params (joined into the report by =trial.number=).

* Troubleshooting
:PROPERTIES:
:CUSTOM_ID: sec-troubleshooting
:END:

** =ModuleNotFoundError: No module named 'conf'=

You ran =pytest …= directly instead of =python -m pytest …=.  Use
the latter — it puts the project root on =sys.path= so
=conf.registry= resolves.

** =ConfigAttributeError: Key 'hydra' not in 'Config'=

You called =hydra.compose(…)= without =return_hydra_config=True=.
See =tests/test_init_centering_pilot.py:test_pilot_sweep_composes_via_cli=
for the right pattern.

** =Unknown cell '…'= from the launcher

The cell name must match =cell_name(form, mode, tracking)=.  Examples
of the 20 legal names:

#+begin_src bash
uv run python -c "
from experiments.init_centering.launch_phase_d import all_phase_d_cells
for n in all_phase_d_cells(): print(n)
"
#+end_src

** Optuna DB collisions

If two CLI invocations share =hydra.sweeper.study_name=, Optuna
treats them as one study (the new trials append to the existing
DB).  This is desirable for resuming a sweep but surprising
otherwise — use a timestamped =--study-prefix= (e.g.
=phase_d_$(date +%Y%m%d)=) for fresh runs.

** =auto-degenerate= warning

Setting =baseline_form=zero baseline_mode=learnable= triggers a
warning + clamps =baseline_mode → pinned=.  That's the intended
behaviour (=zero= has no learnable parameters); the warning is the
UX cue.  The launcher's preset list skips the degenerate cells
entirely.

** Eval is slow (~90s for 5 steps of training)

Phase-A's =stage2_elbo_surrogate= dominates eval time on CPU.
That's the cost of computing the full multi-term ELBO over the val
loader, not a smoke-specific issue.  Eval cost is roughly constant
per cell regardless of training step count — for sweep planning,
expect ~20s per trial of pure eval overhead on a GPU machine.
